Inicio rápido

Las criptomonedas han reducido drásticamente la barrera para aceptar pagos en la web. BlockPays hace que sea más fácil que nunca aceptar la criptomoneda de la forma en que las criptomonedas fueron diseñadas para ser aceptadas: de una manera verdaderamente de igual a igual.


Introducción

BlockPays le permite ser su propio banco con todos los beneficios de un servicio alojado. Ya no es necesario construir y mantener infraestructura para monitorear la cadena de bloques; manténgase enfocado en administrar su negocio y déjenos el resto a nosotros. Al usar claves públicas creadas en el registro para cada criptomoneda, podemos generar direcciones de pago en su nombre y monitorear continuamente la cadena de bloques para detectar cuándo se realizan los pagos.

Los clientes ahora pueden pagarle directamente desde su computadora o dispositivo móvil usando blockchain como red de liquidación. Con las criptomonedas ya no necesita recopilar y almacenar credenciales de pago o información confidencial del cliente.


Pagos de criptomonedas

Los pagos actuales dependen de que los clientes llenen formularios con información de tarjeta de crédito. Esta información se utiliza para extraer pagos directamente del cliente. Las criptomonedas son diferentes. En cambio, una billetera se usa para enviar pagos directamente al comerciante. Un cliente especifica la cantidad de criptomoneda junto con una dirección antes de enviar fondos. De esta manera, los pagos de criptomonedas son pagos push.

Cuando un cliente solicita pagar con criptomoneda, creamos un cargo que representa el pago esperado. Un cargo puede considerarse como una solicitud de pago en una o más criptomonedas. Dado que los pagos de criptomonedas se realizan a través de una red separada, se genera una dirección de pago única por criptomoneda en la creación de cargos para que podamos asociar clientes a sus pagos. Cuando un cliente realiza un pago de criptomonedas, genera una transacción que luego se transmite a la red de criptomonedas para su validación / confirmación.

Una vez que se crea un cargo, comenzamos a monitorear estas direcciones únicas en las redes respectivas para detectar cualquier pago entrante. Cada cargo tiene un estado de pago asociado
